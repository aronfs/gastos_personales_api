# gastos-personales-api

**Runtime**: Bun (not Node). All commands use `bun run` / `bunx`.

## Quick start

```bash
cp .env.example .env
docker compose up -d postgres       # start PostgreSQL 17 on :5432
bun install
bun run prisma:generate             # generate Prisma client
bun run prisma:migrate              # apply dev migrations
bun run prisma:seed                 # creates admin + demo users/seed data
bun run dev                         # http://localhost:3000
```

## Architecture

Express app with 4-layer separation:

```
Route (zod validation) → Controller → Service → Repository → Prisma
```

- **Entry**: `src/server.ts` → `src/app.ts`
- **Routes**: `src/routes/*.routes.ts` — each mounts under `/api/v1/<resource>`
- **Validators**: `src/validators/*.validator.ts` — Zod schemas; validated via `validate` middleware
- **Controllers**: `src/controllers/*.controller.ts` — thin, catch errors, call service
- **Services**: `src/services/*.service.ts` — business logic, throws `{ statusCode, message }`
- **Repositories**: `src/repositories/*.repository.ts` — Prisma queries only
- **Middleware**: `src/middlewares/` — `auth.middleware.ts` (JWT), `role.middleware.ts`, `error.middleware.ts`, `validation.middleware.ts`

## Key conventions

- **Env config**: validated at startup via Zod in `src/config/env.ts`. `DATABASE_URL` required; JWT secrets ≥ 32 chars.
- **API prefix**: `/api/v1` (configurable via `API_PREFIX` env)
- **Response helpers**: `src/utils/response.ts` — `sendSuccess`, `sendCreated`, `sendError`, `sendNotFound`, `sendUnauthorized`, `sendForbidden`, `sendBadRequest`, `sendConflict`
- **Error handler** (`error.middleware.ts`): handles Prisma errors (P2002 conflict, P2025 not found, P2003 FK), JWT errors, generic 500. Dev env exposes error details.
- **Auth**: Bearer JWT access token via `authenticate` middleware. Refresh token rotation. Roles and permissions system.
- **Pagination**: `src/utils/pagination.ts` — `parsePagination(page, limit)`, `buildPaginationMeta(total, page, limit)`. Max limit 100.
- **Path alias**: `@/*` → `src/*` via tsconfig paths (Bun may not resolve this; use relative imports).
- **No tests**: no test framework or test files exist.
- **Bruno collection**: `bruno/` — API request collection for Bruno (REST client).
- **Swagger**: at `/api-docs` when `SWAGGER_ENABLED=true`.

## Database

- PostgreSQL via Prisma 6 (schema: `prisma/schema.prisma`)
- Migrations: `prisma/migrations/`
- Seed creates: admin (`admin@gastos.local` / `Admin123*`), demo user (`user@gastos.local` / `User123*`), roles/permissions, categories, incomes, expenses.
- Prisma client config at `prisma.config.ts` (auto-generated by Prisma 6).
- Prisma singleton in `src/config/database.ts` — prevents hot-reload connection leak.

## Docker

`docker compose up` starts `postgres:17-alpine` + builds/runs the API. DB connection auto-configures via `DATABASE_URL`.

## Preserve

- `src/generated/prisma` is gitignored — Prisma output directory.
- `no-console: warn` in ESLint — prefer `logger` from `src/utils/logger.ts` (winston).
